# Level 1 Acceptance Criteria

- [ ] Freighter is detected and can be authorized.
- [ ] A connected G-address is displayed.
- [ ] Disconnect clears the application wallet session.
- [ ] Wrong-network users are instructed to switch to Testnet.
- [ ] Native XLM balance is fetched from Horizon and displayed.
- [ ] An unfunded wallet can request Testnet XLM from Friendbot.
- [ ] Recipient address is validated.
- [ ] Amount is positive and has no more than seven decimals.
- [ ] The transaction is signed inside Freighter.
- [ ] The signed transaction is submitted to Stellar Testnet.
- [ ] Success displays a transaction hash and explorer link.
- [ ] Rejection or network failure displays a clear error.
- [ ] Lint, type check, tests, and production build pass.
- [ ] README includes setup instructions and screenshots.
